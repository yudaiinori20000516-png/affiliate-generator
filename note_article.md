正直、最初にこの環境構築をやったとき、「Node.jsって何？」「npmって何？」状態だったんで、同じような人でも絶対つまずかないように書きます。コピペだけで進められるようにしてあるんで、1個ずつやっていってください。

## 【ステップ1】環境構築 ── パソコンに「土台」を作る

まず最初にやることは、プログラムを動かすための土台をパソコンに入れること。料理でいうと「まな板と包丁を揃える」みたいなフェーズです。

### Node.jsを入れる

Node.jsっていうのは、JavaScriptっていうプログラミング言語をパソコン上で動かすためのソフトです。ブラウザの中だけじゃなくて、パソコン上でプログラムを実行できるようにしてくれるやつ。これがないと何も始まらないんで、最初に入れます。

**Macの場合（ターミナルを開いて以下をコピペ）：**

```bash
brew install node
```

もし「brewが入ってない」と言われたら、先にこっちを実行してください。Homebrew（ホームブルー）っていう、Mac用のソフト管理ツールです。

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

**Windowsの場合：**

ターミナルじゃなくて、ブラウザで以下のURLを開いてください。

```
https://nodejs.org/ja
```

開いたら「LTS」って書いてあるほう（左側の緑のボタン）をクリックしてダウンロード。あとはインストーラーの指示に従って「次へ」を連打するだけです。途中の設定は全部デフォルトのままでOK。

**ちゃんと入ったか確認：**

Mac・Windows共通で、ターミナル（Windowsの場合はコマンドプロンプトかPowerShell）を開いて以下を打ってください。

```bash
node -v
```

`v20.18.0` みたいにバージョン番号が出たら成功です。数字は多少違っても大丈夫。

### プロジェクトフォルダを作る

次に、今回の自動化プログラム一式を入れておくフォルダを作ります。ターミナルで以下を順番にコピペしてください。

```bash
mkdir ai-fukugyou
cd ai-fukugyou
npm init -y
```

1行目で「ai-fukugyou」っていうフォルダを作って、2行目でそのフォルダの中に入って、3行目でプロジェクトの初期設定ファイルを自動生成しています。`npm init -y`を実行すると`package.json`っていうファイルが勝手にできます。これはプロジェクトの設定書みたいなものです。

**ここで1つ大事な作業があります。** 今作った`package.json`をテキストエディタ（メモ帳でもVS Codeでも何でもいい）で開いて、`"main": "index.js"` の下あたりに以下の1行を追加してください。

```json
"type": "module",
```

これを入れないと、あとで出てくるコードの`import`っていう書き方がエラーになります。自分はこれを入れ忘れて30分くらいハマりました（笑）

### 必要なパッケージをインストールする

パッケージっていうのは、他の人が作ってくれた便利な部品みたいなものです。ゼロから全部自分で書くのは無理なんで、既にある部品を組み合わせて使います。以下をターミナルでコピペ実行。

```bash
npm install @anthropic-ai/sdk twitter-api-v2 playwright
```

それぞれ何をしてくれる部品かというと、

**@anthropic-ai/sdk** → Claude（AIのほう）に文章を書いてもらうための部品
**twitter-api-v2** → X（旧Twitter）に自動で投稿するための部品
**playwright** → ブラウザを自動操作するための部品。Noteへの自動投稿に使う

インストールが終わったら、Playwrightが使うブラウザもダウンロードしておく必要があります。

```bash
npx playwright install chromium
```

### .envファイルを作る

.env（ドットイーエヌブイ）ファイルっていうのは、パスワードとかAPIキーとか、人に見せちゃいけない情報をまとめて書いておくファイルです。プロジェクトフォルダの中に`.env`という名前のファイルを作って、以下の内容をコピペしてください。

```env
ANTHROPIC_API_KEY=ここにAnthropicのAPIキーを貼る
RAKUTEN_APP_ID=ここに楽天のアプリIDを貼る
RAKUTEN_AFFILIATE_ID=ここに楽天のアフィリエイトIDを貼る
RAKUTEN_ACCESS_KEY=ここに楽天のアクセスキーを貼る
MOSHIMO_A_ID=ここにもしものa_idを貼る
X_API_KEY=ここにXのAPIキーを貼る
X_API_SECRET=ここにXのAPIシークレットを貼る
X_ACCESS_TOKEN=ここにXのアクセストークンを貼る
X_ACCESS_TOKEN_SECRET=ここにXのアクセストークンシークレットを貼る
NOTE_EMAIL=ここにNoteのログインメールアドレスを貼る
NOTE_PASSWORD=ここにNoteのログインパスワードを貼る
THREADS_ACCESS_TOKEN=ここにThreadsのアクセストークンを貼る
THREADS_USER_ID=ここにThreadsのユーザーIDを貼る
```

**「=」の後ろにスペースは入れないでください。** `ANTHROPIC_API_KEY=sk-ant-xxxxx` みたいにイコールの直後からキーを書きます。

あと、.envファイルは**絶対にSNSやブログに公開しないでください。** パスワードが全部入ってるんで。Gitを使う人は`.gitignore`に`.env`を追加しておくこと。

### 各APIキーの取得場所一覧

「APIキーって何？どこで手に入るの？」って思いますよね。APIキーっていうのは、外部サービスを使うための「合言葉」みたいなものです。各サービスに「自分のプログラムから使わせてね」と申請すると発行してもらえます。

**Anthropic（Claude AIのAPIキー）**
→ https://console.anthropic.com/
アカウント作成後、左メニューの「API Keys」から発行。クレジットカード登録が必要。最初は5ドル分くらい入金しておけば十分です。

**楽天アフィリエイト（APP_ID / AFFILIATE_ID / ACCESS_KEY）**
→ https://webservice.rakuten.co.jp/
楽天会員でログインして「アプリ登録」。アプリを1個作ると3つのキーが全部もらえます。AFFILIATE_IDは楽天アフィリエイトのトップページ（https://affiliate.rakuten.co.jp/）にログインすると自分のIDが表示されてます。

**もしもアフィリエイト（a_id）**
→ https://af.moshimo.com/
会員登録して、楽天市場の提携を申請。承認されたら「広告リンク取得」のページに行くと、URLの中に`a_id=数字`って入ってるので、その数字がa_idです。

**X（旧Twitter）のAPIキー4種類**
→ https://developer.x.com/
開発者アカウントを申請。Freeプランでも投稿はできます。プロジェクトを作ってアプリを登録すると、API Key / API Secret / Access Token / Access Token Secretの4つが発行されます。**権限を「Read and Write」にするのを忘れずに。** デフォルトだとRead onlyになってて投稿できないです。

**Note（メールアドレスとパスワード）**
→ 普段Noteにログインしてるメールアドレスとパスワードをそのまま.envに書くだけ。APIキーの発行は不要です。ブラウザ自動操作でログインするんで。

**Threads（アクセストークンとユーザーID）**
→ https://developers.facebook.com/
Meta（Facebook）の開発者アカウントが必要。アプリを作成して、Threads APIの権限を追加。Graph APIエクスプローラーでアクセストークンを発行。ユーザーIDはThreadsプロフィールページのURLに入ってる数字です。ここだけ正直ちょっと面倒なんで、Threads投稿は後回しにして先にXとNoteだけ動かすのもありです。

## 【ステップ3：楽天アフィリエイトAPIの設定】

ここからが収益化の核心部分。楽天の商品データを自動で引っ張ってきて、自分のアフィリエイトリンク（紹介リンク）を付けて投稿する仕組みを作る。

### 楽天デベロッパーアカウントの作り方

まず楽天のAPI（外部からデータを取得する窓口みたいなもの）を使うために、開発者登録をする。

1. https://webservice.rakuten.co.jp/ にアクセス
2. 右上の「新規登録」→楽天会員IDでログイン（なければ作る）
3. ログイン後「アプリ登録」をクリック
4. アプリ名は何でもいい。自分は「my-affiliate-bot」にした
5. URLも自分のNoteのURLとかで大丈夫

登録が終わると、管理画面に**applicationId**（アプリID）と**affiliateId**（アフィリエイトID）が表示される。この2つをメモしておく。

ちなみにaccessKeyはapplicationIdと同じものを指してる場合がほとんど。ドキュメントによって呼び方が違うだけで、混乱する必要はない。

### もしもアフィリエイトのa_idの取得方法

「なんで楽天に直接じゃなく、もしも経由にするの？」って思うと思う。理由は単純で、**もしも経由のほうが振込手数料が無料**だから。楽天アフィリエイトだと楽天ポイントで支払われることが多いけど、もしも経由なら現金で銀行振込してくれる。

1. https://af.moshimo.com/ でアカウント作成
2. ログイン後「プロモーション検索」で「楽天市場」と検索
3. 「楽天市場の商品購入」という案件があるので「提携申請」する（即承認される）
4. 提携後、「広告リンク取得」画面を開く
5. そこに表示されるURLの中に `a_id=XXXXXXX` という数字がある。これがa_id

### 楽天URLをもしも経由に変換する仕組み

自動生成するリンクの形式はこうなる。

```
https://af.moshimo.com/af/c/click?a_id=YOUR_A_ID&p_id=54&pc_id=54&pl_id=616&url=楽天商品URLをURLエンコード
```

`p_id=54&pc_id=54&pl_id=616` は楽天市場を示す固定値。変える必要なし。

URLエンコードっていうのは、URLに含まれる特殊文字（`&`とか`=`とか）を別の表記に置き換えること。JavaScriptだと `encodeURIComponent()` って関数を使えば一発で変換できる。

```javascript
const rakutenUrl = "https://item.rakuten.co.jp/xxxxx/yyyyy/";
const mosimoUrl = `https://af.moshimo.com/af/c/click?a_id=${process.env.MOSHIMO_A_ID}&p_id=54&pc_id=54&pl_id=616&url=${encodeURIComponent(rakutenUrl)}`;
```

.envファイルに以下を追加しておく。

```
RAKUTEN_APP_ID=あなたのapplicationId
RAKUTEN_AFFILIATE_ID=あなたのaffiliateId
MOSHIMO_A_ID=あなたのa_id
```

## 【ステップ4：X（Twitter）自動投稿の設定】

### X Developer Portalでのアプリ作成手順

1. https://developer.x.com/en/portal/dashboard にアクセス
2. Xアカウントでログイン
3. 「Sign up for Free Account」を選択（無料プラン）
4. 利用目的を英語で書く欄がある。「I want to post automated tweets about tech and productivity」とか適当で大丈夫
5. アプリが作成されたら、「Keys and Tokens」タブを開く

### 無料tierでできること・できないこと

ここ、自分がハマったポイント。**無料プランだと月1,500件の投稿ができる**。1日50件くらいの計算。自分の用途だと1日4〜6投稿なので余裕。

ただし**リプライ（返信）をAPIから送ろうとすると403エラー（権限がないという意味）で弾かれる**。リプライ機能を使いたいならBasicプラン（月$200）が必要。正直、高すぎるので諦めた。自動投稿だけなら無料で十分。

### APIキーの取得と.envへの設定

「Keys and Tokens」画面で以下の4つを生成する。

1. **API Key**（Consumer Keyとも呼ばれる）
2. **API Key Secret**（Consumer Secretとも呼ばれる）
3. **Access Token**
4. **Access Token Secret**

Access TokenとSecretは「Generate」ボタンを押すと表示される。**一度しか表示されない**ので、必ずその場でコピーすること。忘れたらRevoke（無効化）して再生成すればいいけど、面倒。

.envに追加する。

```
X_API_KEY=あなたのAPI Key
X_API_SECRET=あなたのAPI Key Secret
X_ACCESS_TOKEN=あなたのAccess Token
X_ACCESS_SECRET=あなたのAccess Token Secret
```

**大事な設定がもう1つ**。Portalの「User authentication settings」で、App permissionsを「Read and Write」に変更する。デフォルトだとReadだけになってて、投稿しようとしてもエラーになる。自分はこれに気づくまで2時間溶かした。

## 【ステップ5：Threads自動投稿の設定】

### Meta Developersでのアプリ作成

1. https://developers.facebook.com/ にアクセス
2. Metaアカウント（Instagramアカウント連携済み）でログイン
3. 「マイアプリ」→「アプリを作成」
4. ユースケースで「その他」→タイプは「ビジネス」を選択
5. アプリ名を適当に入れて作成
6. ダッシュボードから「Threads API」を探して「セットアップ」をクリック

### アクセストークンの取得

Threads APIのトークン取得は正直ちょっと面倒。普通にブラウザでOAuth認証（外部サービスにログイン許可を出す仕組み）を通す必要がある。

自分はこれを簡略化するスクリプトを作った。プロジェクト内の `get-threads-token.js` を使う。

```bash
node get-threads-token.js
```

実行するとブラウザが開いて、Threadsアカウントへの認証画面が出る。許可すると、ターミナル（黒い画面）にアクセストークンが表示される。これをコピー。

### THREADS_USER_IDの取得方法

トークンを取得した後、以下のコマンドでユーザーIDを取れる。

```bash
curl "https://graph.threads.net/v1.0/me?access_token=あなたのトークン"
```

返ってきたJSONの中の `id` がユーザーID。

.envに追加。

```
THREADS_ACCESS_TOKEN=あなたのトークン
THREADS_USER_ID=あなたのユーザーID
```

### 投稿形式について

Threadsでは**スレッド形式（連続投稿）**で投稿している。

- **1投稿目**：記事の要約テキスト（「1/2」と付ける）
- **2投稿目**：NoteのURLだけを貼る（「2/2」と付ける）

なぜ分けるかというと、**URLを含む投稿はThreadsのアルゴリズムで表示が抑えられがち**だから。テキストで興味を引いてから、次の投稿でリンクを出す。この形式にしてからクリック率が体感で2倍くらいになった。

## 【よくあるトラブルQ&A 10問】

ここからは、僕が実際にハマったトラブルを全部まとめた。たぶん、同じところで詰まる人が多いと思う。エラーが出たらまずここを見てほしい。

---

**Q1: 楽天APIで商品が見つからない**

原因：検索キーワードが具体的すぎる。「パナソニック ナノケア EH-NA0J ピンクゴールド 2024年モデル」みたいに長いと、楽天API側でヒットしない。

解決策：**キーワードは商品名＋ブランド名くらいに絞る**。「ナノケア パナソニック」程度でいい。Claude Codeに「検索キーワードを短く整形してからAPIに投げて」と指示しておくと安定する。

---

**Q2: X投稿でリプライが403エラーになる**

原因：Xの開発者ポータルで、アプリの権限が「Read」のままになっている。投稿するには「Read and Write」が必要。

解決策：developer.x.comにログインして、アプリの設定画面で**User authentication settingsを「Read and Write」に変更**する。変更したあと、アクセストークンを再発行するのを忘れずに。僕はこれで丸一日潰した。

---

**Q3: Noteの投稿が途中で止まる**

原因：Playwrightっていうブラウザ自動操作ツールが、Noteのページ読み込みを待ちきれずにタイムアウトしている。Noteは読み込みが遅いときがある。

解決策：waitForTimeoutの値を**5000（5秒）から10000（10秒）に増やす**。Claude Codeに「Noteの操作部分のタイムアウトを10秒に変更して」と言えばやってくれる。それでもダメなら15秒。回線が遅い時間帯を避けるのも手。

---

**Q4: サムネイル生成でエラーが出る**

原因：日本語フォントがサーバーに入っていない。文字が□□□みたいな豆腐になるか、そもそもエラーで落ちる。

解決策：Ubuntu（VPSのOS）なら`sudo apt install fonts-noto-cjk`を実行する。これでGoogleが作った日本語フォントが入る。**フォントを入れたあとスクリプトを再起動**すれば直る。

---

**Q5: Threads投稿が反映されない**

原因：Threads APIは投稿を「作成」したあと「公開」する、2段階の手順が必要。作成だけして公開してないパターンが多い。

解決策：スクリプト内で**creation_idを取得→それを使ってpublishのAPIを叩く**、という2ステップになっているか確認する。Claude Codeに「Threads投稿のpublishステップが抜けてないか確認して」と聞けば一発でわかる。

---

**Q6: .envの読み込みがうまくいかない**

原因：.envファイルの書き方ミス。イコールの前後にスペースを入れてたり、値をクォーテーションで囲んでいたりする。

解決策：.envは**`API_KEY=abc123`のようにスペースなし、クォーテーションなし**で書く。あとdotenvパッケージがインストールされてるかも確認。`npm list dotenv`で出てこなければ`npm install dotenv`を実行する。

---

**Q7: Claude APIのmax_tokensで記事が途中で切れる**

原因：max_tokensっていうのは「AIが返す文章の最大文字数（正確にはトークン数）」の上限設定のこと。初期値が1024とかだと、記事の途中でブツッと終わる。

解決策：**max_tokensを4096に設定する**。日本語だと大体3000〜4000字くらいの記事が生成できる。Claude Codeに「max_tokensを4096にして」と伝えるだけ。ただし増やすほどAPI料金も上がるので、必要十分な値にしておく。

---

**Q8: もしもアフィリリンクが無効になっている**

原因：提携申請がまだ承認されていないか、a_id（自分のアフィリエイトID）が間違っている。

解決策：もしもアフィリエイトの管理画面で、**楽天との提携ステータスが「提携中」になっているか確認**する。a_idは「プロモーション検索→楽天→広告リンク取得」の画面に表示されている数字。コピペミスが多いから注意。

---

**Q9: Playwrightが起動しない**

原因：Playwrightのブラウザ本体がインストールされていない。npmでplaywrightパッケージを入れただけでは動かない。

解決策：**`npx playwright install chromium`を実行**する。VPSの場合はさらに`npx playwright install-deps`も必要。これでChromiumっていうブラウザの本体と、それが動くために必要なソフトがまとめて入る。

---

**Q10: APIの費用が気になる**

原因：気になるよね。僕も最初ビビってた。

解決策：実際の内訳を書くと、**Claude API（Haiku）が月500〜800円、楽天APIは無料、X APIは無料プラン、Threads APIも無料**。合計で月1000円いかない。1日2記事ペースでこの金額。5万円稼いで経費1000円なら十分すぎる。最初はHaikuモデルで始めて、稼げてからSonnetに上げればいい。

## 【もしもアフィリエイトとの連携方法】

ここが収益化の肝。楽天アフィリエイトを直接使うんじゃなくて、**もしもアフィリエイト経由で楽天の商品を紹介する**のがポイント。

**登録から提携申請まで**

1. もしもアフィリエイト（af.moshimo.com）にアクセスして無料会員登録する
2. 登録したら「プロモーション検索」で「楽天市場」と検索する
3. 楽天市場の案件が出てくるので「提携申請する」をクリック
4. 楽天は**即時提携**なので、申請した瞬間から使える

**a_idの確認方法**

提携したら「広告リンク取得」を押す。そこに表示されるURLの中に`a_id=数字`っていう部分がある。**この数字が自分専用のアフィリエイトID**。これをスクリプトの.envファイルに`MOSHIMO_A_ID=数字`として保存しておく。

**toMoshimoLink()関数の説明**

パート2で作ったスクリプトの中に、楽天の商品URLをもしも経由のURLに変換する関数がある。やっていることは単純で、楽天の商品URLをエンコード（URLに使える形に変換）して、もしものリダイレクトURL`https://af.moshimo.com/af/c/click?a_id=自分のID&p=商品URL`の形に組み立てているだけ。

読者がこのリンクをクリックして楽天で買い物すると、**もしもアフィリエイト経由で自分に報酬が入る**仕組み。

**なぜ楽天直より「もしも経由」がお得か**

楽天アフィリエイトを直接使うと、報酬は楽天ポイントで支払われる。しかも現金化が面倒。

もしも経由だと、まず**報酬が現金で銀行振込**される。さらに「W報酬制度」っていうのがあって、**通常報酬に加えて12%のボーナスが上乗せ**される。つまり楽天から1000円の報酬が発生したら、もしもからさらに120円もらえる。同じことをやるなら、もしも経由にしない理由がない。
