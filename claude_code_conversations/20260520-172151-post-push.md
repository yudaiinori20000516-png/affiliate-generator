## 変更レビュー

### 1. 変更の意図
一貫している。大きく2本立て：
- **generate.js強化**: 設計シート参照・有料記事化・記事品質向上（文字数倍増、図解・注釈追加）
- **Codex→ClaudeCode通知基盤**: push後に自動でレビューを依頼する仕組み（今まさに動いている）

### 2. 運用上のリスク（要注意）

**[重大] 有料記事マーカーが未処理のまま本文に残る**  
`generate.js` の `postToNote()` は `【ここから有料部分｜100円】` というマーカーをプロンプトで生成させているが、Note投稿時にそのマーカーを解釈して有料設定するコードが存在しない。  
→ 現状では「`【ここから有料部分｜100円】`という文字列がそのまま記事本文に残って無料公開される」可能性が高い。

**[中] max_tokens 3000→6000 でAPI費用が約2倍**  
記事品質向上のトレードオフとして許容範囲だが、記事数を増やすと顕著になる。

**[軽微] strategy_output.mdのアフィリエイト候補に「Claude Pro（Anthropic公式プラン）」が残っている**  
楽天で検索できないキーワードなのでgenerate.jsのルールで弾かれるはずだが、そのままCodexに渡すと他のスクリプト（gen-premium-article.js）では弾かれない。

### 3. Note/X/Threads投稿フローへの影響

| 媒体 | 影響 |
|---|---|
| Note | 有料設定未対応のまま投稿フローが走る。**未確認のまま本番投稿するのは危険** |
| X | テンプレ禁止ルールが強化されたので品質向上。問題なし |
| Threads | 変更なし |

`gen-premium-article.js` との役割が重複してきている（あちらは500円、こちらは100円）。整理が必要になりそう。

### 4. 次にCodexへ依頼すべき改善

**優先度高:**
1. `postToNote()` 内で `【ここから有料部分｜100円】` マーカーを検出し、Note UIの有料設定（ラジオボタン押下・価格入力）を自動化するPlaywright処理を追加する
2. 本文投稿前にマーカー文字列を本文から除去して、文字がそのまま表示されないようにする

**優先度中:**
3. `generate.js`（100円）と `gen-premium-article.js`（500円）の責務分担をCLAUDE.mdに明記する
4. `claude_code_strategy_output.md` の「アフィリエイト候補キーワード」から楽天非対応の候補（Claude Pro等）を削除 or コメントアウトする
