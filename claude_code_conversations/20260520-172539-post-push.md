## 変更内容レビュー

### 1. 変更の意図 — 自然か

**概ね自然。** 既存の `gen-premium-article.js` と異なり、`generate.js`（無料記事フロー）でも本文に `【ここから有料部分｜100円】` マーカーを埋め込めば、100円有料記事として投稿できるようにした変更。マーカーを取り除いてからNoteに投稿するロジックも一貫している。

---

### 2. 運用上のリスク

**高めの懸念点:**

- **セレクタが脆弱。** `text=有料` や `input[placeholder*="価格"]` はNote UIが更新されると即座に壊れる。有料設定に失敗しても `console.warn` だけで投稿が続行されるため、**気づかずに無料で公開されるリスク**がある。
- **価格がハードコード。** `DEFAULT_PAID_PRICE = "100"` が定数になっており、100円以外にしたい場合はコード変更が必要。マーカーに価格を含める形（例: `【ここから有料部分｜500円】`）にすれば柔軟性が増す。
- **`PAID_MARKER` の除去がreplace（非グローバル）。** 同マーカーが本文中に2回以上あっても1回目しか消えない（バグの種）。

**低めの懸念点:**

- `preparePaidArticleBody` が `generate.js` にしか存在しないため、`gen-premium-article.js` 側との二重管理になり得る。

---

### 3. Note/X/Threads フローへの影響

- **X・Threads には影響なし。** マーカー除去前の `body` を使っているわけではなく、X/Threads 投稿はこの関数を経由しないため問題ない。
- **Note フローのみ変更あり。** `noteBody`（マーカー除去済み）を本文入力に使う変更は正しい。

---

### 4. Codexへの次の依頼候補

優先度順:

1. **有料設定失敗時に投稿を止める** — `console.warn` を `throw` に変えて、気づかず無料公開されるのを防ぐ
2. **マーカーのグローバル replace 化** — `.replace(PAID_MARKER, "")` → `.replace(new RegExp(PAID_MARKER, 'g'), "")`
3. **マーカーから価格を動的に読み取る** — `【ここから有料部分｜500円】` のような形式をパースして価格も可変にする
4. **`preparePaidArticleBody` を共通モジュールに切り出す** — `gen-premium-article.js` との重複排除
