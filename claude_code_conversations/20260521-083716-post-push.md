## 変更サマリと評価

### 変更の意図（自然か？）

**意図は明確で自然。** `gen-premium-article.js` と `generate.js` に重複していた有料記事ロジックを `paid-note.js` に抽出したリファクタリング。合わせて2点の動作改善も含まれている。

**動作変更①（重要）** — 投稿順序が変わった

```
変更前: X投稿 → Note投稿 → Threads投稿
変更後: Note投稿 → X投稿 → Threads投稿
```

Threads投稿がNote URLを使うため、Noteを先に投稿するのは**正しい修正**。

**動作変更②** — 有料設定失敗時の挙動がサイレントスキップから例外（強制停止）に変更。無料で誤公開するリスクが排除されており**安全性向上**。

---

### リスク・懸念点

1. **`PAID_MARKER_PATTERN` を `/g` フラグつきでモジュールレベル定数にしている**
   - `matchAll()` との組み合わせは問題ない
   - ただし将来 `.test()` や `.exec()` で使われると `lastIndex` が汚染されるリグレッションリスクあり
   - 軽微だが `/gu` フラグへの変更か、使用箇所でインライン生成が望ましい

2. **`toPublicNoteUrl` のフォールバックがサイレント**
   - マッチしない場合（下書き確認ページ等）は生URLを返す設計
   - Threadsに下書きURLが投稿されても気づけない
   - `console.warn` か `throw` があった方が安全

3. **`MAX_PAID_PRICE = 9999` と運用ルール（500〜980円）のズレ**
   - CLAUDE.mdの「500〜980円」と乖離しており、誤って高額設定した場合に検知できない

4. **`repost-premium.js` の重複は解消されたか未確認**
   - 同様のロジックが残っている可能性
