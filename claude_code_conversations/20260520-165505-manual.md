### 2. 運用上のリスク

**[高] 有料記事マーカーがNote UIに反映されない**
`generateArticle()` が `【ここから有料部分｜100円】` を本文テキストとして埋め込むが、`postToNote()` の Playwright フローに**有料/無料切り替えボタンをクリックする処理がない**。マーカーは単なるプレーンテキストとして投稿されてしまい、Note の有料ゲートが機能しない。

**[中] max_tokens=6000 × バッチ処理のコスト増**
`titles.txt` に複数タイトルがある場合、1記事あたり claude-opus-4-6 で最大 6000 tokens。10記事で旧比2倍のコストになる。大量バッチ前に確認推奨。

**[低] `claude_code_strategy_output.md` が未コミット**
`loadStrategyForTitle()` は `fs.existsSync` でガード済みなので動作は問題ない。ただし `??` (untracked) のままなので、別マシンや CI では常に strategy なしで動く。

**[低] `scripts/` ディレクトリが未コミット**
`notify:claude` / `push:notify` スクリプトは `scripts/` 参照だが、`git status` では `??` (untracked)。`npm run notify:claude` が他環境で失敗する。
